# Project Notes

This project is designed to replace an older online voting system repository with a more recruiter-friendly full-stack project.

## Why this project is stronger

- It aligns with software engineer, full-stack engineer, backend engineer, and finance/FinTech roles.
- It uses common production web application skills: React, Node.js, Express.js, REST APIs, MongoDB, Docker, and CI.
- It connects well with backend/payment and financial workflow experience.
- It is easier to explain in interviews than a simple class-style CRUD app.

## Talking points for interviews

- How portfolio summary metrics are calculated from holdings.
- How transactions are modeled separately from current holdings.
- How user-specific data can be scoped by authenticated user ID.
- How the project can be extended with JWT, live market data, and AWS deployment.
- Why REST API design and service-layer calculations make the app easier to maintain.
